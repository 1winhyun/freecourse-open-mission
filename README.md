# freecourse-open-mission

# 2, 3주차 미션, 새로운 게임을 KOTLIN으로 개발

<hr>
<hr>

## 공통 설계 원칙

- 입력/출력과 도메인 로직 분리
  - Input/Output 클래스는 I/O만 담당합니다.
  - 비즈니스 로직은 Service, 검증은 Validator에서 처리합니다.

- 랜덤 의존성 분리
  - RandomGenerator, NumberGenerator 인터페이스를 통해 랜덤 생성을 분리합니다.
  - 서비스는 인터페이스만 의존하여 테스트가 쉽습니다.
- 예외 기반 검증
  - 잘못된 입력은 IllegalArgumentException을 던집니다.
  - Controller/Input 레벨에서 예외 메시지를 출력 후 재입력 받습니다.
- 상수 분리
  - 메시지 및 에러 문구, 숫자 범위는 object 상수로 관리합니다.
  - 예: Errors, Messages, Numbers, Alphabets

## 테스트 전략
- 각 게임(Lotto / RacingCar / GuessGame)의 핵심 도메인/서비스/검증 로직을 단위 테스트로 작성했습니다.
- 랜덤성은 인터페이스로 분리하고, 테스트 전용 구현체를 사용하여 작성했습니다.

## 프로젝트 이름: 3가지 게임 모음

- 주제: 세 가지 콘솔 게임(Lotto, RacingCar, GuessGame)을 하나의 애플리케이션으로 제공하는 프로젝트입니다. 사용자는 프로그램 실행 시 번호를 입력해 실행할 게임을 선택할 수 있습니다.
  - 1: Lotto
  - 2: RacingCar
  - 3: GuessGame

## 1. 초간단 자동차 경주 게임 Kotlin 구현
### 기능 설명
- 주어진 횟수 동안 n대의 자동차는 전진 또는 멈출 수 있다.
- 각 자동차에 이름을 부여할 수 있다. 전진하는 자동차를 출력할 때 자동차 이름을 같이 출력한다.
- 자동차 이름은 쉼표(,)를 기준으로 구분하며 이름은 5자 이하만 가능하다.
- 사용자는 몇 번의 이동을 할 것인지를 입력할 수 있어야 한다.
- 전진하는 조건은 0에서 9 사이에서 무작위 값을 구한 후 무작위 값이 4 이상일 경우이다.
- 자동차 경주 게임을 완료한 후 누가 우승했는지를 알려준다. 우승자는 한 명 이상일 수 있다.
- 우승자가 여러 명일 경우 쉼표(,)를 이용하여 구분한다.
- 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException을 발생시킨 후 애플리케이션은 종료되어야 한다.

### 주요 예외 처리
- 금액
  - 1000원 미만 → 에러
  - 1000원 단위가 아님 → 에러
- 로또 번호
  - 6개가 아니면 → 에러
  - 1~45 범위를 벗어나면 → 에러
  - 중복 숫자 존재 시 → 에러
- 당첨/보너스 번호
  - 당첨 번호 6개가 아닐 경우 → 에러
  - 보너스 번호 미입력 / 여러 개 입력 시 → 에러
  - 보너스 번호가 당첨 번호와 중복될 경우 → 에러

## 2. 로또 발매기 Kotlin 구현
### 기능 설명
- 로또 숫자 범위는 1 ~ 45
- 1개의 로또 발행 시 중복되지 않는 6개의 숫자 뽑기
- 당첨 번호 추첨 시 미중복 숫자 6개 + 보너스 번호 1개
- 당첨은 1등 ~ 5등까지 존재.
  - 1등: 6개 일치, 2,000,000,000원
  - 2등: 5개 + 보너스 일치, 30,000,000원
  - 3등: 5개 일치, 1,500,000원
  - 4등: 4개 일치, 50,000원
  - 5등: 3개 일치, 5,000원
- 로또 구입 금액 입력 시 구입 금액에 해당하는 만큼 로또 발행
- 로또 1장은 1,000원
- 당첨 번호와 보너스 번호 입력받기
- 구매한 로또 번호와 당첨 번호를 비교해 당첨 내역 및 수익률 출력 및 게임 종료
- 사용자가 잘못된 값 입력 시, IllegalArgumentException 발생
  - "[Error]"로 시작하는 에러 메시지 출력 후 그 부분부터 재입력 받기

### 주요 예외 처리
- 자동차 이름
  - 공백 혹은 빈 문자열 → 에러
  - 5글자 초과 → 에러
  - 한 대도 없으면 → 에러
- 시도 횟수
  - 미입력 → 에러
  - 숫자가 아님 → 에러
  - 1 미만 → 에러

## 3. 숫자와 영어 맞추기 게임 Kotlin 구현
### 기능 설명
- 숫자 버전과 영어 버전 중 하나를 선택하여 게임을 시작
  - 영어: A ~ z 
  - 숫자: 1 ~ 100
- 숫자 버전: 난수를 생성
- 영어 버전: 대문자와 소문자 사이의 알파벳을 임의로 선택
  - 난수 사용
- 사용자의 추측 값을 입력
- 사용자의 입력값과 정답을 비교하여 UP or DOWN을 판단하여 출력
- 사용자가 정답을 맞추면 “정답!” 문자열과 시도한 횟수를 출력

### 주요 예외 처리

- 게임 타입
  - 숫자가 아닌 입력 → 에러
  - 1, 2 이외의 숫자 → 에러
- 숫자 추측
  - 숫자가 아닌 입력 → 에러
  - 현재 허용된 범위(min~max) 밖의 숫자 → 에러
- 알파벳 추측
  - 입력이 1자가 아님 → 에러
  - 알파벳이 아님 → 에러
  - 허용된 범위 밖의 알파벳 → 에러
